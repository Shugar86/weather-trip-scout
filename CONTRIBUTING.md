# Участие в Weather Trip Scout

Спасибо, что заглянул! Weather Trip Scout — персональный open-source проект. PR приветствуются, но если планируешь большое изменение — сначала обсуди его в issue, чтобы не делать двойную работу.

---

## Быстрый старт для контрибьютора

1. Сделай форк репозитория.
2. Создай ветку с понятным именем:

   ```bash
   git checkout -b feature/short-description
   ```

3. Внеси изменения. Придерживайся минимального diff и KISS.
4. Запусти проверки:

   ```bash
   source .venv/bin/activate
   make check
   ```

5. Убедись, что `.env`, ключи и токены не попали в индекс:

   ```bash
   git status
   ```

6. Обнови `README.md`, `AGENTS.md` и `CHANGELOG.md`, если менялась архитектура, конфигурация или команды.
7. Открой Pull Request с кратким описанием:
   - Что изменилось.
   - Зачем.
   - Как проверял.

---

## Стиль кода

- Python 3.12+, type hints, `mypy --strict`.
- Максимальная длина строки: 88 символов (`ruff`).
- Импорты сортируются автоматически через `ruff`.
- Функции должны быть короткими и делать одну вещь.
- Провайдеры реализуют `Protocol` из соответствующего `base.py`.
- Сервисы stateless; зависимости передаются через конструктор.

## Коммиты

Рекомендуется [Conventional Commits](https://www.conventionalcommits.org/):

```text
feat: add new weather provider
test: cover scoring edge cases
docs: update README with Docker notes
fix: handle empty candidate list
```

---

## Definition of Done для PR

- `make check` проходит без ошибок.
- Новая логика покрыта тестами или обоснованно исключена.
- Документация отражает изменения.
- Коммит-сообщения понятные.
- В PR нет секретов, API-ключей и личных конфигураций.

---

## Поведение

- Будь вежливым и конструктивным.
- Не коммить секреты, API-ключи и личные конфигурации.
