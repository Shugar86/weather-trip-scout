# Weather Trip Scout — контракт для агента

## 0. Режим работы

- **pair-programmer**, не автономный продакт. Пользователь ведёт приоритеты; агент исполняет, проверяет, сужает неопределённость.
- **Язык:** русский (продукт для русскоязычного Telegram-окружения).
- **Тон:** `calm` — тихий утренний советчик.

## 1. Продукт

Weather Trip Scout — асинхронный Python-агент, который каждое утро анализирует погоду в радиусе до 100 км от домашней точки и присылает в Telegram подборку лучших направлений для поездки: текст + статическая карта.

Ключевые ценности:

- Автоматизация рутины «куда поехать сегодня».
- Честность: если погода никуда не годится, агент так и скажет.
- Минимальная зависимость от платных ключей: основные провайдеры бесплатны.

## 2. Вайб продукта

- **Вайб:** `calm` — «тихий утренний советчик».
- **3 правила:**
  1. **Честность прежде всего.** Нет хороших направлений — сообщаем об этом прямо, без попыток «подогнать» результат.
  2. **Никакого хайпа.** Формулировки спокойные, деловые, но человечные; без капса, эмодзи-шторма и маркетинговых словечек.
  3. **Бережное отношение к данным и API.** По умолчанию используем бесплатные провайдеры (Open-Meteo, Overpass, staticmap+OSM); платные fallback включаются только при наличии ключа.

## 3. Стек и архитектура

| Область | Технология |
|---------|------------|
| Язык | Python 3.12+ |
| Конфигурация | Pydantic v2, pydantic-settings, PyYAML |
| Погода | Open-Meteo (primary), OpenWeatherMap (fallback) |
| Гео-поиск | Overpass API / OpenStreetMap |
| Карты | `staticmap` + OSM; Mapbox — опционально |
| Telegram | `python-telegram-bot` |
| Тестирование | pytest, pytest-cov, pytest-asyncio |
| Линтинг / типы | ruff, mypy `--strict` |
| Автоматизация | Makefile, Docker, systemd |

```text
app/
├── config/           # Pydantic-модели для config.yaml и .env
├── core/             # Базовые исключения
├── domain/           # Чистые dataclass-модели и scoring-конфиг
├── providers/        # Protocol-based адаптеры
│   ├── weather/      # Open-Meteo, OpenWeatherMap
│   ├── geo/          # Overpass / OSM
│   └── maps/         # staticmap+OSM, Mapbox
├── services/         # Stateless бизнес-логика
├── jobs/             # Сценарии запуска
└── main.py           # CLI entrypoint
```

## 4. Инженерная дисциплина

- **KISS / минимальный diff / YAGNI.** Не усложняй без явной необходимости.
- **Не ломай:** если не уверен на 100% — спроси, не делай предположений.
- **Не коммить `.env`, ключи, токены, пароли, `*.pem`, `id_*`, `node_modules/`, `__pycache__/`.**
- **Секреты только в `.env`**, конфигурация — в `config.yaml`.
- **Провайдеры** реализуют `Protocol` из `providers/<area>/base.py`.
- **Сервисы** stateless, принимают провайдеры через конструктор и возвращают доменные модели.
- **Job** (`jobs/morning_report.py`) связывает сервисы и провайдеры в единый сценарий.
- **Типизация строгая:** `mypy --strict`.

## 5. Команды

Все команды выполняй внутри виртуального окружения (`source .venv/bin/activate`).

```bash
make install    # установить зависимости
make run        # одиночный отчёт
make report     # то же самое
make test       # pytest + coverage
make lint       # ruff + mypy
make check      # lint + test
make format     # ruff format и автофикс
```
